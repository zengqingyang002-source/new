# 20 分钟演示脚本

## 0-2 分钟：项目介绍

本项目叫“基于 Dify 的国际教育智能服务平台”，服务于国际教育机构的客户咨询、客户研判、员工协同和学生服务场景。

技术架构为 Vue3 前端、FastAPI 后端、MySQL 数据库和 Dify 智能体。

## 2-4 分钟：首页仪表盘

登录管理员账号：

```text
admin / 123456
```

展示：

- 意向客户总数
- 待处理投诉
- 心理预警
- 项目分布
- 业务演示主线

## 4-7 分钟：客服咨询

进入客服咨询页，提问：

```text
新加坡 2+2 国际本科班多少钱？
```

说明：

- 回答来自 Dify 知识库。
- 知识库包含公司信息、项目资料、政策指南和 FAQ。
- 如果 Dify 不可用，后端 fallback 保证演示稳定。

## 7-11 分钟：客户画像研判

进入客户研判页，点击“填入样例”，内容类似：

```text
张三，男，19岁，高中毕业，家庭经济条件较好，想出国读本科，英语一般，家长希望费用不要太高，优先考虑新加坡方向。
```

点击“研判并入库”。

展示输出：

- 推荐项目
- 新加坡分数
- 德国分数
- A/B/C 等级
- 匹配原因
- 缺失信息
- 销售建议

说明：

客户研判不是纯大模型判断，而是“大模型抽取 + 后端规则评分 + Dify 生成建议”的组合。

## 11-13 分钟：CRM 跟进

进入客户管理 CRM。

展示：

- 新增客户已经保存。
- 可以查看客户详情。
- 可以把状态更新为“跟进中”或“已签约”。

## 13-15 分钟：企业助手和日报

进入企业助手。

提问：

```text
有哪些投诉还没有处理？
```

再演示日报提交。

说明：

企业助手面向员工，解决客户查询、新人指南、日报、投诉和请假处理问题。

## 15-17 分钟：学生服务

进入学生服务。

演示：

- 提交请假。
- 员工审批请假。
- 提交投诉。
- 员工处理投诉。
- 查看心理预警。

## 17-19 分钟：智能报告

进入智能报告页。

生成：

- 客户经营分析报告
- 投诉处理周报

说明：

报告通过后端汇总数据库数据，再交给 Dify Workflow 生成管理层可读的总结。

## 19-20 分钟：总结亮点

总结三点：

1. 业务闭环完整：咨询、研判、CRM、学生服务、报告。
2. AI 有实际价值：知识库问答、客户画像、销售建议、智能报告。
3. 工程结构完整：前端、后端、数据库、Dify、Docker、文档齐全。
