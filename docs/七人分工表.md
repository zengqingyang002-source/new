# 七人分工表

| 成员 | 角色 | 负责内容 |
| --- | --- | --- |
| 1 | 组长 / 产品 / 答辩负责人 | 需求整理、功能范围、流程图、PPT、演示节奏 |
| 2 | 前端 A | 登录页、首页仪表盘、客服咨询页 |
| 3 | 前端 B | 客户研判、CRM、学生服务、智能报告页面 |
| 4 | 后端 A | FastAPI 架构、MySQL、登录鉴权、基础 CRUD |
| 5 | 后端 B | 文件解析、客户研判规则、请假投诉、报告接口 |
| 6 | Dify A | 知识库整理、客服 Chatflow、新人指南问答 |
| 7 | Dify B | 客户研判 Workflow、企业助手、智能报告 Workflow |

## 协作建议

1. 每个人负责一个可演示模块。
2. 组长统一收口演示流程。
3. Dify 配置人员需要和后端同学确认 API Key 和接口格式。
4. 前端同学优先保证页面可跑通，再做美化。
5. 后端同学优先保证接口稳定，Dify 不可用时要有 fallback。
